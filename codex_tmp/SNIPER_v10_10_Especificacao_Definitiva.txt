3  Feature Engines: FracDiff em log(preço), Weight 
Cutoff τ=1e-5, Expanding Window 
DOIS ERROS CORRIGIDOS NESTA PARTE (v10.6 + v10.7) 
v10.6: weight cutoff τ=1e-5 em vez de window=10 — preserva memória de longo prazo real 
(~500 obs efetivos vs 10). v10.7: FracDiff aplicada sobre log(preços), não preços brutos — 
coeficientes da série de Taylor aplicados em variações relativas (log-retornos) invariantes à 
escala, não em magnitudes de dólares que criam heterocedasticidade artificial. 
 
3.1  Por que FracDiff deve ser aplicada em log(preço) 
O erro matemático silencioso do espaço linear 
BTC em $3.000 em 2019. BTC em $70.000 em 2024. Fator de escala: 23×. 
Um movimento de 5% em 2019 = $150. Um movimento de 5% em 2024 = $3.500. 
Os coeficientes da série de Taylor (w_1, w_2, ...) são ESCALARES aplicados sobre VALORES 
ABSOLUTOS em dólares. 
No espaço linear: a variância de 2024 domina completamente os dados de 2019. A série 
diferenciada é heterocedástica por construção — viola a premissa de estacionariedade que a 
FracDiff devia garantir. 
No espaço log: log(70000) - log(3000) = 3.15. Um movimento de 5% em qualquer época = 
log(1.05) ≈ 0.049. Invariante à escala. 
Regra: fracdiff_series recebe SEMPRE np.log(prices), nunca prices brutos.

def fracdiff_log(prices: np.ndarray, d: float, tau: float = 1e-5) -> np.ndarray: 
    """ 
    Diferenciacao fracionaria NO ESPACO LOGARITMICO. 
  
    CORRETO (v10.7):   fracdiff_log(np.log(prices), d, tau) 
    INCORRETO (v10.6): fracdiff_log(prices, d, tau)  -- magnitudes absolutas em 
dolares 
  
    Recebe: log_prices = np.log(prices) -- variacao percentual invariante a escala 
    Retorna: serie com NaN nos primeiros (len(weights)-1) pontos. 
    """ 
    log_p = np.log(prices)  # garantia: sempre converter para log AQUI 
    w     = fracdiff_weights(d, tau=tau) 
    n_w   = len(w) 
    n     = len(log_p) 
    result = np.full(n, np.nan) 
    for t in range(n_w - 1, n): 
        result[t] = np.dot(w, log_p[t - n_w + 1 : t + 1]) 
    return result

4  Filtro de Regime: Winsorização → RobustScaler → 
PCA Walk-Forward → HMM 
PROBLEMA CORRIGIDO v10.8: PCA Frágil a Outliers (Norma L2) 
StandardScaler divide pelo desvio padrão GLOBAL — que em crypto inclui eventos de 10-20σ 
(Março 2020: retorno diário −50%, FTX Novembro 2022: volatilidade +800%). PCA minimiza 
erro L2 (eleva ao quadrado): um único crash rotaciona violentamente PC1 para apontar na 
direção do evento extremo, não na estrutura de volatilidade contínua. O HMM subsequente 
recebe componentes principais contaminados por cisnes negros isolados. Correção: 
Winsorização 1%-99% antes do scaler + RobustScaler (mediana + IQR) em vez de 
StandardScaler. 
 
4.1  Pipeline Robusto: Winsorização → RobustScaler → PCA 
Por que StandardScaler falha em crypto 
FTX crash (Nov 2022): funding_rate_ma7d atingiu +15σ em um único dia. basis_3m colapsa 
−12σ. 
StandardScaler(X): (X - mean) / std. O std inclui esses outliers — todos os outros pontos ficam 
comprimidos em um intervalo minúsculo. 
PCA em X_scaled: os autovetores são calculados minimizando Σ(erro)². Erro ao quadrado 
amplifica o outlier de 15σ por fator 225×. 
Resultado: PC1 aponta na 'direção do colapso FTX'. HMM aprende estados que são na 
verdade 'FTX vs. não-FTX', não 'bull vs. bear'. 
RobustScaler: (X - median) / IQR. A mediana e o IQR são imunes a valores extremos. Outliers 
de 15σ não distorcem a transformação. 
Winsorização 1%-99% ANTES do scaler: cap explícito nos extremos. Garante que mesmo um 
outlier de 50σ seja tratado como o percentil 99.

from sklearn.decomposition import PCA 
from sklearn.preprocessing import RobustScaler 
from hmmlearn.hmm import GaussianHMM 
import numpy as np, pandas as pd 
  
HMM_FEATURES = [ 
    'ret_1d','ret_5d','realized_vol_30d','vol_ratio', 
    'funding_rate_ma7d','basis_3m','stablecoin_chg30', 
    'btc_ma200_flag','dvol_zscore' 
] 
N_PCA_COMPONENTS    = 2 
MIN_VARIANCE_EXPLAINED = 0.80 
WINSOR_LOW  = 0.01   # percentil 1% 
WINSOR_HIGH = 0.99   # percentil 99% 
  
def winsorize_array(X: np.ndarray, 
                    low_pct: float = WINSOR_LOW, 
                    high_pct: float = WINSOR_HIGH) -> np.ndarray: 
    """ 
    Winsorização feature-wise: cap em percentis low_pct e high_pct. 
    Calculados SOMENTE nos dados de treino. Aplicados em treino e teste. 
  
    Garante que eventos de 50 sigma sejam tratados como percentil 99. 
    Nao elimina o evento -- apenas limita seu impacto no PCA. 
    """ 
    X_clip = X.copy() 
    for col in range(X.shape[1]): 
        lo = np.percentile(X[:, col], low_pct * 100) 
        hi = np.percentile(X[:, col], high_pct * 100) 
        X_clip[:, col] = np.clip(X[:, col], lo, hi) 
    return X_clip 
  
def fit_robust_pca(train_features: np.ndarray, 
                   n_components: int = N_PCA_COMPONENTS) -> tuple: 
    """ 
    Pipeline robusto: Winsorização -> RobustScaler -> PCA. 
    Tudo fittado SOMENTE nos dados de treino (expanding window). 
  
    v10.7: StandardScaler -> PCA (fragil a outliers) 
    v10.8: Winsorização + RobustScaler -> PCA (robusto a cisnes negros) 
  
    RobustScaler: centra pela mediana, escala pelo IQR. 
    Imune a outliers de 10-20 sigma (FTX, Março 2020). 
  
    Retorna (winsor_bounds, scaler, pca, var_explained). 
    """ 
    # 1. Calcular bounds de winsorização no treino

def transform_robust_pca(features: np.ndarray, 
                           winsor_bounds: list, scaler, pca) -> np.ndarray: 
    """ 
    Transforma features NOVAS com pipeline fittado no treino. 
    Aplica os mesmos bounds de winsorização do treino (nunca re-calcular). 
    """ 
    X_clip = features.copy() 
    for col, (lo, hi) in enumerate(winsor_bounds): 
        X_clip[:, col] = np.clip(features[:, col], lo, hi) 
    X_scaled = scaler.transform(X_clip) 
    return pca.transform(X_scaled)

5  Triple-Barrier: Detecção HLC, Slippage 
√(ordem/volume), P&L Microestrutural Real 
CORREÇÃO v10.10: Market Impact Linear → Lei da Raiz Quadrada 
v10.9 usou exit_price = barrier_sl − pavio × impact_factor (linear). Modelos Kyle (1985) e 
Almgren-Chriss (2001) estabelecem: impacto de mercado ∝ √(ordem/volume). Liquidez esgota 
de forma sub-linear: dobrar o tamanho da ordem não dobra o slippage, mas multiplicá-lo por √2 
sim. Em flash crash (volume[t] colapsado), √(order_size/volume[t]) cresce rapidamente — 
capturando o custo real de forçar capital através de funil de liquidez estreito. 
 
5.1  Fundamento: Por que a Lei da Raiz Quadrada 
Da microestrutura à fórmula aplicada 
Kyle (1985): λ = impacto permanente por unidade de ordem. Preço move λ×Q onde Q é o 
volume da ordem. 
Almgren-Chriss (2001): custo de execução = σ × √(Q/V) onde V é o volume de mercado 
disponível. 
Empírico: meta-análise de Bouchaud et al. (2010) em 12+ mercados confirma: ΔP ∝ σ × √(Q/V). 
Em flash crash: V colapsa (market makers retiram quotes), Q/V explode → slippage se torna 
severo mesmo para ordens pequenas. O fator linear captura isso de forma inadequada. 
Interpretação: √(Q/V) é a 'participação de mercado fracionária'. Para Q/V = 1% → fator 0.10; 
para Q/V = 10% → fator 0.316 (não 0.10 × 10 = 1.0 como seria linear).

import numpy as np, pandas as pd 
  
def compute_intraday_vol(high_prices: pd.Series, low_prices: pd.Series, 
                          ewm_span: int = 20) -> pd.Series: 
    """ 
    Volatilidade intradiária estimada como (High - Low) / Close (Parkinson). 
    Proxy para sigma_intraday sem dados tick. 
    """ 
    hl_range = (high_prices - low_prices) / ((high_prices + low_prices) / 2.0) 
    return hl_range.ewm(span=ewm_span, min_periods=5).mean()

6  Clustered Feature Importance: Variation of 
Information 
CRÍTICA A — Incorporada v10.4: Correlação Linear é Ingênua para LightGBM 
O clustering hierárquico do CFI v10.3 usava distância baseada em correlação de Pearson. 
LightGBM explora relações não-lineares. Duas features com correlação zero podem ter Mutual 
Information alta — são redundantes para o modelo mas o clustering linear as trata como 
independentes. Variation of Information (VI) detecta redundância de qualquer tipo: linear, não-
linear, threshold, polinomial. É o único framework com garantia matemática de consistência (VI 
é métrica própria — satisfaz desigualdade triangular). Referência: Lopez de Prado, AFML 2018, 
Cap. 3. 
 
6.1  Fundamento Matemático da VI 
Mutual Information e Variation of Information 
Mutual Information: I(X;Y) = H(X) + H(Y) - H(X,Y) 
I(X;Y) captura toda dependência informacional entre X e Y — linear E não-linear. 
Variation of Information: VI(X,Y) = H(X|Y) + H(Y|X) = H(X,Y) - I(X,Y) 
VI = 0 → X e Y são funcionalmente idênticas (redundância total). VI alta → informativamente 
independentes. 
Pearson = 0 NÃO implica independência informacional. VI = 0 implica. Esta é a diferença que 
importa para o LightGBM. 
 
6.2  Implementação da VI e Clustering 
import numpy as np, pandas as pd 
from sklearn.preprocessing import KBinsDiscretizer 
  
def variation_of_information(x: np.ndarray, y: np.ndarray, n_bins: int = 10) -> 
float: 
    """ 
    VI(X,Y) normalizada em [0,1]. 
    0 = features informativamente idênticas | 1 = informativamente independentes. 
    n_bins adaptativo: min(10, sqrt(N)) — reduz sensibilidade com N pequeno. 
    Referência: Lopez de Prado, AFML 2018, Cap. 3. 
    """ 
    n = len(x) 
    bins = max(5, min(n_bins, int(np.sqrt(n)))) 
    disc = KBinsDiscretizer(n_bins=bins, encode='ordinal', strategy='quantile')

    xd = disc.fit_transform(x.reshape(-1,1)).ravel().astype(int) 
    yd = disc.fit_transform(y.reshape(-1,1)).ravel().astype(int) 
  
    def entropy(z): 
        _, cnts = np.unique(z, return_counts=True) 
        p = cnts / cnts.sum() 
        return -np.sum(p * np.log2(p + 1e-12)) 
  
    hx  = entropy(xd) 
    hy  = entropy(yd) 
    hxy = entropy(xd * (bins+1) + yd)   # joint via pairs 
    mi  = hx + hy - hxy 
    vi  = hx + hy - 2*mi 
    return max(vi / max(hxy, 1e-12), 0.0)  # normalizada, não negativa 
  
def compute_vi_distance_matrix(feature_df: pd.DataFrame, n_bins: int = 10) -> 
pd.DataFrame: 
    """Matriz de distância VI para todas as features. O(n_features^2).""" 
    feats = feature_df.columns.tolist(); n = len(feats) 
    D = np.zeros((n,n)) 
    for i in range(n): 
        for j in range(i+1,n): 
            v = variation_of_information(feature_df[feats[i]].values, 
                                          feature_df[feats[j]].values, n_bins) 
            D[i,j]=D[j,i]=v 
    return pd.DataFrame(D, index=feats, columns=feats)

PROBLEMA CORRIGIDO v10.7: Stacking Não-Ortogonal 
O meta-modelo usa P_bma (predição do modelo primário) como feature de entrada. Se P_bma 
é gerado por walk-forward simples e o CPCV do meta-modelo embaralha blocos temporais, 
P_bma[t] pode ter sido gerado por um modelo que viu o período correspondente ao fold de 
teste do CPCV. O meta-modelo aprenderia a assinatura temporal das predições do primário — 
vazamento de segunda ordem. Correção: gerar P_bma via Purged K-Fold sobre TODO o 
dataset antes do meta-modelo. Cada P_bma[t] gerado por um fold que nunca viu t. 
 
8.1  Stacking Ortogonal: P_bma via Purged K-Fold 
Por que walk-forward simples não é suficiente para stacking 
Walk-forward gera P_bma[t] treinando o primário com dados até t−1. Parece correto. 
Problema: o CPCV do meta-modelo cria folds que EMBARALHAM o dataset. O fold de teste 
pode incluir datas próximas a datas de treino do modelo primário. 
Consequência: P_bma para o período de teste do CPCV foi gerado por um primário que 'viu' 
dados vizinhos a esse período. A assinatura temporal de P_bma vaza informação sobre quando 
a predição foi feita. 
Purged K-Fold sobre todo o dataset: P_bma[t] gerado pelo fold que exclui a janela [t−embargo, 
t+embargo]. Cada observação nunca viu a si mesma. Folds alinhados e expurgados. O meta-
modelo recebe predições genuinamente OOS. 
Referência: Lopez de Prado, AFML 2018, Cap. 7 (Stacking em séries temporais financeiras).

    Cada P_bma[t] e gerado pelo fold k que exclui a janela embargada em torno de

teste 
        test_min, test_max = test_idx.min(), test_idx.max()

def compute_label_uniqueness_dynamic(triple_barrier_df: pd.DataFrame) -> 
pd.Series: 
    """Unicidade usando duração real do trade via t_touch (v10.6). Sem 
mudancas.""" 
    t0 = pd.to_datetime(triple_barrier_df.index).values.astype(np.int64) 
    t1 = pd.to_datetime(triple_barrier_df['t_touch']).values.astype(np.int64) 
    overlap     = (t0[None,:] <= t1[:,None]) & (t1[None,:] >= t0[:,None]) 
    concurrency = overlap.sum(axis=1).astype(float) 
    return pd.Series(1.0/np.maximum(concurrency,1.0), 
index=triple_barrier_df.index) 
  
def calculate_effective_N_dynamic(triple_barrier_df): 
    u=compute_label_uniqueness_dynamic(triple_barrier_df); N=u.sum() 
    if N<60: m="LOGISTIC_ONLY" 
    elif N<120: m="LGBM_STRICT" 
    else: m="LGBM_STANDARD" 
    return N,u,m

def _compute_time_decay_weights(dates: pd.DatetimeIndex, 
                                 halflife_days: int = 180) -> np.ndarray: 
    """ 
    Pesos de decaimento temporal exponencial. 
    Observacoes recentes recebem peso proximo a 1.0. 
    Observacoes com halflife_days atras recebem peso 0.5. 
    Observacoes com 2 x halflife_days atras recebem peso 0.25. 
  
    Garante que a calibracao isotonica reflita a eficiencia de mercado ATUAL, 
    nao a media de 5 anos de microestrutura diferente. 
  
    halflife_days=180: seis meses. Ajustar para mercados mais estaticos. 
    halflife_days=90: tres meses. Para crypto em regime de alta volatilidade. 
    """ 
    max_date  = dates.max() 
    days_ago  = (max_date - dates).days.astype(float) 
    decay_cst = np.log(2) / halflife_days   # lambda para halflife 
    weights   = np.exp(-decay_cst * days_ago) 
    return weights / weights.sum() * len(weights)   # normalizar: media = 1.0

    bin_edges = np.linspace(0, 1, n_bins + 1) 
    ece = 0.0 
    for i in range(n_bins): 
        mask = (probs >= bin_edges[i]) & (probs < bin_edges[i+1]) 
        if mask.sum() == 0: continue 
        ece += (mask.sum()/len(probs)) * abs(probs[mask].mean() - 
labels[mask].mean()) 
    return ece

10  Position Sizing: f_base = E[ganho] / CVaR-99% ×

PROBLEMA CORRIGIDO v10.7: Kelly Plug-in Subestima Tail Risk 
Kelly numérico com parâmetros estimados pela distribuição histórica de P&L subestima 
sistematicamente o risco de cauda em crypto: retornos leptocúrticos (fat tails), saltos de 
Poisson (crashes repentinos como LUNA, FTX), e liquidez inexistente em gaps extremos. Se o 
pior cenário do backtest foi −15%, mas o pior cenário real pode ser −40% (gap de liquidez não 
observado), o Kelly otimizará para sobreviver ao −15% e falhar no −40%. Correção: E[ganho] / 
CVaR_99% como base do Kelly. Se choques extremos mostram −30%, o numerador é limitado 
matematicamente. 
 
10.1  Fundamento: f_base via CVaR-99% 
Fórmula completa v10.7 
CVaR_99% = Expected Shortfall no percentil 1% = média dos 1% piores retornos do P&L 
histórico da estratégia 
f_cvar_base = max(E[P&L], 0) / CVaR_99%  (CVaR em valor absoluto) 
f_quarter = 0.25 × f_cvar_base  (quarter-Kelly conservador sobre base já CVaR-limitada) 
Φ(z) = CDF normal em z = (P_meta − 0.5) / sqrt(P_meta × (1 − P_meta)) 
Tamanho final = f_quarter × Φ(z(P_meta)) × HRP_weight × capital 
Se CVaR_99% = 30% (estratégia perde 30% nos 1% piores dias) e E[P&L] = 2%, f_cvar_base 
= 2%/30% = 6.7% 
Quarter-Kelly: 0.25 × 6.7% = 1.67%. Escalado por Φ(z): se P_meta=0.80 → 1.67% × 0.76 = 
1.27% 
Referência: Lopez de Prado, AFML 2018, Cap. 13 — bet sizing com tail risk.

def pmeta_scaling_factor(p_meta: float) -> float: 
    """Fator CDF normal para escalar Kelly por P_meta (v10.5 — inalterado).""" 
    p = float(np.clip(p_meta, 0.501, 0.999)) 
    z = (p - 0.5) / np.sqrt(p * (1 - p)) 
    return float(norm.cdf(z))

                               returns_history: pd.DataFrame, 
                               kelly_fractions: dict, 
                               capital_total: float,

    """ 
    CVaR de portfólio conjunto para os ativos com sinais ativos. 
    Executado ANTES de enviar ordens à Binance. 
  
    active_signals: lista de ativos com sinal no dia 
    hrp_weights: {ativo: peso_hrp} 
    returns_history: DataFrame (N_dias x N_ativos) de retornos históricos diários 
    kelly_fractions: {ativo: f_kelly_cvar_escalado} -- frações individuais já 
calculadas

    portfolio_cvar_limit: CVaR máximo do portfólio conjunto (padrão: 15%)

    if not active_signals: 
        return {} 
  
    # Construir vetor de pesos do portfólio ativo 
    ativo_weights = {} 
    for ativo in active_signals:

import pandas as pd, numpy as np

class CircuitBreakerCorwinSchultz: 
    """ 
    Circuit Breaker v10.10: Corwin-Schultz Spread + Funding Rate. 
  
    SUBSTITUI o OFI de barras OHLCV (v10.9) que era ~90% ruido branco. 
    OFI genuino requer dados tick-by-tick (L1) para classificacao bid/ask. 
    Calcular OFI de barras de 4h usando Close vs Open e simplesmente ruido. 
  
    Corwin-Schultz (2012): extrai spread implícito bid-ask de High/Low de 
    barras adjacentes. Nao requer dados tick. Funciona com OHLCV diario ou 4h. 
    Um pico no spread implícito sinaliza que market makers retiraram liquidez 
    do book -- exatamente o que acontece nas primeiras horas de um colapso. 
  
    Combinado com Funding Rate extremo (independente, nao requer L1). 
    """ 
    def __init__(self, 
                 sigma_threshold : float = 3.0, 
                 window_bars     : int   = 6,    # 6 barras de 4h = 24h de janela 
                 history_bars    : int   = 90,   # 90 barras de referencia (15d em 
4h) 
                 halt_duration_h : int   = 24): 
        self.sigma_threshold = sigma_threshold 
        self.window_bars     = window_bars 
        self.history_bars    = history_bars 
        self.halt_duration   = halt_duration_h 
        self.halt_active     = False 
        self.halt_until      = None 
  
    def corwin_schultz_spread(self, high: np.ndarray, low: np.ndarray) -> 
np.ndarray: 
        """ 
        Estimador de spread bid-ask de Corwin e Schultz (2012). 
  
        Usa o ratio (H/L) de barras adjacentes para estimar o spread implícito. 
        Intuitivo: se H/L e muito grande em relacao ao dia anterior, os market 
        makers ampliaram o spread — sinal de retirada de liquidez. 
  
        high, low: arrays de precos (N barras). Retorna spread estimado (N-1 
valores). 
  
        Referencia: Corwin e Schultz (2012), "A Simple Way to Estimate Bid-Ask 
        Spreads from Daily High and Low Prices", Journal of Finance. 
        """ 
        beta  = np.log(high[1:] / low[1:])**2 + np.log(high[:-1] / low[:-1])**2 
        gamma = np.log(np.maximum(high[1:], high[:-1]) / 
                       np.minimum(low[1:], low[:-1]))**2 
        alpha = (np.sqrt(2 * beta) - np.sqrt(beta)) / (3 - 2*np.sqrt(2)) - 
np.sqrt(gamma / (3 - 2*np.sqrt(2))) 
        spread = 2 * (np.exp(alpha) - 1) / (1 + np.exp(alpha)) 
        return np.clip(spread, 0, 1)   # spread nao pode ser negativo

def deflated_sharpe_ratio(sr_backtest,sr_benchmark,n_trials,n_obs,skew,kurt): 
    from scipy.stats import norm 
    E=((1-np.euler_gamma)*norm.ppf(1-1/n_trials)+np.euler_gamma*norm.ppf(1-
1/(n_trials*np.e))) 
    sa=sr_backtest*(1-skew*sr_backtest+(kurt-1)/4*sr_backtest**2)**0.5 
    prob=norm.cdf((sa-sr_benchmark)*np.sqrt(n_obs-1)/np.sqrt(1-skew*sa+(kurt-
1)/4*sa**2)) 
    return prob-norm.cdf(E)  # DSR>0: edge real

12.2  DSR Honesto: n_trials do Cemitério de Backtests (v10.8) 
O n_trials=500 é Sub-representação do Viés de Seleção Real 
Surface de parâmetros: 500 combinações finais. Mas todo o processo SNIPER v1.0→v10.8 
envolveu milhares de decisões: features testadas e descartadas, modelos explorados, lógicas 
abandonadas, parâmetros ajustados entre versões. O Sharpe da v10.8 sobreviveu a um 
processo de seleção muito mais amplo do que 500 trials. Usar n_trials=500 no DSR é 
autoengano matemático. Protocolo v10.8: n_trials ≥ 5.000 como lower bound conservador do 
cemitério completo de hipóteses. 
 
from scipy.stats import norm 
import numpy as np 
  
def compute_dsr_honest(sharpe_is: float, n_trials_final: int = 500, 
                        n_trials_total: int = 5000, T: int = 1500, 
                        skewness: float = -0.3, kurtosis: float = 5.0) -> dict: 
    """ 
    DSR com n_trials HONESTO (cemiterio de backtests incluido). 
    Exige DSR > 0.95 para n_trials_total -- nao apenas n_trials_final. 
  
    n_trials_total: estimativa conservadora de TODAS as hipoteses testadas 
    desde v1.0 ate versao atual. Inclui: features descartadas, modelos 
    testados, logicas reescritas, parametros explorados entre versoes. 
    Lower bound recomendado: 5000. 
  
    Referencia: Lopez de Prado, AFML 2018, Cap. 8. 
    """ 
    euler_g = 0.5772 
  
    def sr_star(n_trials): 
        e_max = (1-euler_g)*norm.ppf(1-1/n_trials) + euler_g*norm.ppf(1-
1/(n_trials*np.e)) 
        var_sr = (1 - skewness*1 + ((kurtosis-1)/4)*1) / (T-1) 
        return np.sqrt(var_sr) * e_max 
  
    def dsr(sr_is, n_trials): 
        sr_s  = sr_star(n_trials) 
        var_sr = (1 - skewness*sr_is + ((kurtosis-1)/4)*sr_is**2) / (T-1) 
        return norm.cdf((sr_is - sr_s) / np.sqrt(var_sr)) 
  
    dsr_final = dsr(sharpe_is, n_trials_final) 
    dsr_total = dsr(sharpe_is, n_trials_total)
